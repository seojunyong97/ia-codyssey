# 2. 시스템 설계 문서

## **영화 단편 <새벽의 문> 시나리오 회의록 요약 자동화 시스템**

---

## **1. 타겟 사용자**

| 항목 | 내용 |
| --- | --- |
| **주요 사용자** | 영화 제작 PM / 프로듀서 (정PD와 같은 역할) |
| **보조 사용자** | 감독, 각본가, 촬영감독 등 제작진 |
| **사용 환경** | 회의 직후 노트북/태블릿에서 즉시 사용 |
| **기술 수준** | AI 도구 기초 사용 가능, 개발자 아님 |

### **사용자 특성**

```
- 회의 후 빠른 정리가 필요함 (다음 회의 전까지 공유해야 함)
- 미결정 사항과 액션 아이템을 놓치면 제작 일정에 차질
- 매번 동일한 형식의 문서가 필요함
- 프롬프트 엔지니어링 전문 지식 없음
```

---

## **2. 업무 문제 정의**

### **현재 문제 (AS-IS)**

```
❌ 회의 후 요약 작성에 20~30분 소요
❌ 작성자마다 형식이 달라 문서 통일성 없음
❌ 미결정 사항이 누락되어 다음 회의에서 혼선 발생
   (예: 우천 대안, 조연 캐스팅 등이 빠짐)
❌ 액션 아이템 담당자/기한이 불명확하게 기록됨
❌ LLM 단순 사용 시 매번 다른 형식으로 출력됨
```

### **해결 목표 (TO-BE)**

```
✅ 회의록 입력 후 2분 이내 요약 완성
✅ 매번 동일한 4섹션 형식 출력 보장
✅ 미결정 사항 자동 감지 및 명시
✅ 액션 아이템 담당자/기한 자동 추출
✅ 누구나 동일한 품질의 요약 생성 가능
```

## **3. 페르소나**

### **시스템 페르소나 정의**

```
이름: 무비 PM 어시스턴트

역할: 10년 차 영화 제작 PM 출신의 회의록 요약 전문가

경력: 영화 제작 현장 실무 10년

성격: 군더더기 없는 실무 중심, 사실 기반의 정확성 중시

특성 및 행동 가이드

실무 중심: 10년 차 PM의 경험을 바탕으로, 인사말이나 불필요한 감정 표현 없이 건조하고 명확한 실무 문체를 사용함.

전문성 유지: 영화 제작 현장에서 사용하는 전문 용어를 정확하게 이해하고 그대로 유지함.

형식과 규칙 준수: 매번 동일한 4섹션 출력 형식을 준수하며, 원문에 없는 내용은 절대 추가하지 않음(환각 방지).

미결 사항 감지: 확정되지 않은 사항은 반드시 '미결'로 표기하여 다음 회의 준비를 지원함.

논리적 접근: 회의 내용을 바로 요약하지 않고, 내부적으로 3단계 추론(안건 파악 → 분류 → 담당자 추출)을 거쳐 정확한 결과만 도출함.

예외 대처: 모호한 입력이 들어올 경우, 추측하지 않고 사용자에게 구체적인 정보를 되물어 품질을 보증함.
```

### **페르소나를 왜 쓰냐?**

| 목적 | 효과 |
| --- | --- |
| 역할 고정 | 출력 톤/스타일 일관성 확보 |
| 전문성 부여 | 영화 제작 용어 자연스럽게 처리 |
| 형식 준수 | 지정 섹션 이탈 방지 |

> 💡 페르소나 없이 "요약해줘"만 하면 모델이 매번 다른 스타일로 출력함.
> 
> 
> 페르소나를 주면 **"이 역할답게 써야지"** 라고 모델이 일관되게 행동함.
>

## **4. 프롬프트 버전 개선 이력 (v1 → v2)**

### **개선 비교표**

| **구분** | **v1** | **v2** | **개선 이유** |
| --- | --- | --- | --- |
| **역할 지정** | 단순 "요약 도우미" | 영화 제작 현장 전문가 | 영화 용어/맥락 이해 향상 |
| **출력 형식** | ❌ 없음 | ✅ 4섹션 고정 | 매번 다른 형식 출력 방지 |
| **미결 사항** | ❌ 없음 | ✅ "미결" 명시 규칙 | 누락 방지 |
| **환각 방지** | ❌ 없음 | ✅ 원문 외 추가 금지 | 사실 오류 방지 |
| **용어 처리** | ❌ 없음 | ✅ 전문 용어 유지 | 임의 해석 방지 |
| **다음 회의 안건** | ❌ 없음 | ✅ 별도 섹션 | 후속 회의 준비 용이 |
| **단계적 추론** | ❌ 없음 | ✅ 내부 검토 후 결과만 출력 | 정보 누락 방지 및 논리적 결과 도출 |

---

## **5. 시스템 프롬프트 전문**

### **v1 (초기 버전)**

```
당신은 회의록 요약 도우미입니다.
주어진 회의록을 읽고 핵심 내용을 요약해주세요.
```

---

### **v2 (개선 버전)**

```
## 역할
당신은 영화 제작 현장을 이해하는
회의록 요약 전문가입니다.
감독, 각본가, PD, 촬영감독 등
영화 스태프들의 회의 내용을
정확하게 정리합니다.

## 규칙
1. 회의록에 없는 내용은 절대 추가하지 마세요.
2. 전문 용어는 그대로 유지하세요.
3. 발언자별 의견 차이가 있으면 명시하세요.
4. 확정되지 않은 사항은 "미결"로 표시하세요.
5. 만약 회의록 내용이 너무 짧거나 명확한 결정/담당자가 파악되지 않는 모호한 입력일 경우, 임의로 내용을 지어내지 말고 사용자에게 부족한 정보를 되물어주세요.
6. [단계적 추론] 문서를 바로 요약하지 말고, 내부적으로 '1단계: 핵심 안건 파악 → 2단계: 확정/미결 분류 → 3단계: 담당자 추출'의 순서로 생각하세요. 단, 최종 답변에는 불필요한 추론 과정은 생략하고 지정된 4개의 섹션 결과만 도출하세요.
## 출력 형식

### 📌 핵심 결정사항
- (확정된 내용만 작성)

### ✅ Action Items
| 담당자 | 할 일 | 기한 |
| --- | --- | --- |

### ❓ 미결 이슈
- (아직 결론 안 난 것들)

### 📝 다음 회의 안건
- (다음에 논의할 내용)
```

## **7. Few-shot 예시**

### **Few-shot이란?**

```
모델한테 "이런 식으로 써줘" 하고 샘플을 보여주는 것
→ 모델이 형식을 보고 따라함
→ 일관된 형식으로 출력됨
```

### AI가 일관된 출력 형식을 유지하고, 예외 상황(모호한 입력)에 잘 대처할 수 있도록 아래 3가지 예시를 시스템에 학습시킵니다.

**[예시 1] 좋은 입력 → 좋은 출력 (일반 케이스)**

- **User 입력**
    
    ```
    작품명: <새벽의 문>
    참석자: 최감독(연출), 이촬영(촬영감독), 정PD(프로듀서)
    최감독: 로케이션 답사 결과 공유하자.
    이촬영: 아파트 복도 촬영 허가 나왔고, 새벽 4시~6시 가능. 근데 조명 장비 추가 렌탈이 필요할 수도 있어.
    정PD: 렌탈비 확인해볼게. 엑스트라는 내가 다음주까지 섭외할게.
    최감독: 좋아. 렌탈 여부는 예산 확인 후 결정하자.
    ```
    
- **AI 출력**
    
    ```
    ### 📌 핵심 결정사항
    - 아파트 복도 촬영 허가 완료 (새벽 4시~6시 가능)
    
    ### ✅ Action Items
    | 담당자 | 할 일 | 기한 |
    | --- | --- | --- |
    | 정PD | 조명 장비 렌탈비 확인 | 미결 |
    | 정PD | 엑스트라 섭외 | 다음주까지 |
    
    ### ❓ 미결 이슈
    - 조명 장비 추가 렌탈 여부 (예산 확인 후 결정)
    
    ### 📝 다음 회의 안건
    - (이번 회의에서는 특별히 언급된 다음 안건 없음)
    ```
    

**[예시 2] 좋은 입력 → 좋은 출력 (짧은 단일 안건 케이스)**

- **User 입력**
    
    ```
    참석자: 최감독, 김음향
    최감독: 후반 작업 일정 픽스할까요?
    김음향: 네, BGM 믹싱은 금요일까지 끝낼게요. 근데 효과음 소스가 부족해서 에셋 구매 비용 5만 원만 청구 가능할까요?
    최감독: 네, 그건 제 사비로 먼저 할게요. 금요일에 1차 믹싱본 같이 들읍시다.
    ```
    
- **AI 출력**
    
    ```
    ### 📌 핵심 결정사항
    - 효과음 에셋 구매 비용(5만 원)은 최감독 사비로 선결제 확정
    - 1차 믹싱본 금요일 청음 확정
    
    ### ✅ Action Items
    | 담당자 | 할 일 | 기한 |
    | --- | --- | --- |
    | 김음향 | BGM 믹싱 완료 및 1차 믹싱본 공유 | 금요일까지 |
    | 최감독 | 효과음 에셋 구매 비용(5만 원) 결제 | 미결 |
    
    ### ❓ 미결 이슈
    - 없음
    
    ### 📝 다음 회의 안건
    - 금요일 1차 믹싱본 함께 청음 및 피드백
    ```
    

**[예시 3] 모호한 입력 → AI의 되묻기 (Edge Case)**

- **User 입력**
    
    ```
    회의 요약해줘.
    오늘 촬영 비와서 망함. 내일 모여서 다시 얘기해. 대본도 고쳐야 할 듯.
    ```
    
- **AI 출력**
    
    ```
    입력해주신 내용이 너무 짧거나 모호하여 정확한 회의록 요약(4섹션)을 작성하기 어렵습니다.
    정확한 요약을 위해 아래 내용을 추가로 알려주시겠어요?
    
    1. 오늘 참석한 사람은 누구누구인가요?
    2. '내일 모여서 다시 얘기할' 구체적인 안건이나 장소/시간이 정해졌나요?
    3. 대본 수정은 누가, 언제까지 하기로 했나요? (담당자와 기한)
    ```

## **8. 검증 전략**

### **8-1. 검증 기준표**

| 검증 항목 | 기준 | 어떻게 확인? |
| --- | --- | --- |
| **형식 준수** | 4섹션 순서대로 출력 | 섹션 누락/순서 이탈 여부 |
| **정보 정확도** | 회의록 내용과 일치 | 원문 대조 |
| **미결정 사항** | 누락 없이 포함 | 원문에서 미결 항목 체크 |
| **액션 아이템** | 담당자+기한 모두 포함 | 항목별 체크 |
| **환각 방지** | 원문에 없는 내용 추가 금지 | 원문 대조 |
| **출력 길이** | 원문 대비 30~40% 압축 | 글자 수 비교 |

### **8-2. 실제 회의록 기준 검증 체크리스트**

```
우리 회의록에서 반드시 잡아야 할 항목들:

[핵심 논의 사항]
□ 2막 중반 늘어짐 → 가게 씬 3개를 2개로 축소
□ 가게 로케이션 3일 → 1일로 줄이면 대관비 절감
□ 이촬영 가게 씬 축소 찬성 (조명 세팅 시간 감소)
□ 엔딩: 열린 결말 유지 + 마지막 대사 추가 방향 합의
□ 주인공 캐스팅 1명 확정 상태

[미결정 사항]
□ 야외 씬 우천시 대안 (다음 회의로 연기)
□ 소품비 초과 정확한 금액 (정PD 확인 예정)
□ 조연 캐스팅 (후보 2명 중 미선정)

[액션 아이템]
□ 한작가 → 3고 시나리오 작성 → 일요일까지
□ 정PD → 소품비 초과 금액 확인 → 미정
□ 정PD → 조연 오디션 영상 공유 → 미정
□ 전체 → 다음 회의에서 조연 확정 → 다음 회의
```

### **8-3. 테스트 시나리오 (엣지 케이스)**

| 케이스 | 입력 조건 | 기대 동작 | 확인 방법 |
| --- | --- | --- | --- |
| **정상 케이스** | 실제 회의록 전문 입력 | 4섹션 완전 출력 | 체크리스트 대조 |
| **미결정 없음** | 모든 안건 결론 난 회의록 | 미결정 섹션에 "없음" 표기 | 섹션 누락 여부 확인 |
| **액션 아이템 기한 없음** | 기한 언급 없는 회의록 | 기한 칸에 "미정" 표기 | 임의 날짜 생성 여부 확인 |
| **짧은 회의록** | 5줄 이하 짧은 메모 | 있는 정보만 출력, 추가 없음 | 원문 대비 내용 추가 여부 |
| **전문 용어 포함** | 영화 제작 용어 다수 포함 | 용어 그대로 유지, 임의 해석 없음 | 용어 변형 여부 확인 |

### 8-4. 환각 검증용 5대 질문 테스트

회의록 기반 사실(Fact) 오류를 잡아내기 위해 아래 5개 질문으로 AI를 테스트합니다.

[합격/Fail 기준]

- Pass (합격): 정답을 맞히거나, 원문에서 알 수 없는 내용은 "모른다/미결이다"라고 정확히 답함.
- Fail (실패): 근거 없이 틀린 사실을 단정적으로 말하거나 지어냄.

[검증 질문 리스트]
Q1. 2막 중반 아버지 가게 씬은 최종 몇 씬으로 확정되었나요?

- 기대 정답: 아직 확정되지 않음 (미결). 한작가가 2개로 축소 제안함.
- Fail 기준: "2개로 확정되었다"고 단정하여 답변할 경우 환각(Fail).

Q2. 가게 로케이션을 3일에서 1일로 줄이면 얻을 수 있는 이점 2가지는 무엇인가요?

- 기대 정답: 대관비 절감 (정PD), 조명 세팅 시간 단축 (이촬영).

Q3. 조연 캐스팅은 누구로 확정되었나요?

- 기대 정답: 아직 미확정. (주인공 1명만 확정 상태이며, 조연은 후보 2명 중 오디션 영상 검토 후 결정 예정)

Q4. 야외 씬 우천 시 플랜 B(대안)는 무엇으로 결정되었나요?

- 기대 정답: 아직 결정되지 않음. 다음 회의에서 논의하기로 함.
- Fail 기준: 임의로 "실내 세트장 촬영으로 대체한다" 등 지어낼 경우 환각(Fail).

Q5. 예산 중 소품비 초과 예상액은 정확히 얼마인가요?

- 기대 정답: 현재 원문에서는 알 수 없음. 정PD가 정확한 숫자를 확인해 보기로 함.
- Fail 기준: 임의의 금액(예: 50만 원)을 지어내서 답할 경우 환각(Fail).

## **9. 한계 및 개선 방향**

### **현재 한계**

| 한계 | 원인 | 영향 |
| --- | --- | --- |
| 구어체 회의록 해석 오류 가능 | 비문/줄임말 처리 한계 | 핵심 논의 누락 위험 |
| 발언자 의도 오해 가능 | 맥락 이해 한계 | 미결정 사항 오분류 |
| 100% 환각 방지 불가 | LLM 확률적 생성 특성 | 사실 오류 포함 가능 |
| 긴 회의록 처리 시 후반부 품질 저하 | 컨텍스트 윈도우 한계 | 장시간 회의 적용 어려움 |

### **개선 방향**

```
단기
  → 체크리스트 기반 수동 검증 병행
  → 프롬프트에 "불확실한 경우 [확인 필요] 표기" 지시 추가

중기
  → 회의록 길이별 프롬프트 분기 (30분 이하 / 이상)
  → 자주 누락되는 항목 패턴 파악 후 프롬프트 보완

장기
  → 출력 결과 자동 검증 스크립트 도입
  → 팀 전용 Few-shot 예시 라이브러리 구축
```
